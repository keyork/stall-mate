# 公共厕所坑位选择：大模型决策一致性实验方案

---

## 一、实验总目标

以"公共厕所选坑位"作为决策微任务，系统性检验大模型在以下方面的表现：

1. **决策一致性**：同一问题多次询问，模型是否给出稳定的选择
2. **决策结构性**：模型的选择背后是否存在可推断的决策逻辑
3. **条件敏感性**：当决策条件变化时，模型的选择变化是否合理且可预测
4. **公理符合性**：模型的决策行为是否满足理性决策的基本公理（传递性、独立性等）

实验分为两大阶段：**理想环境实验**（Phase 1）和**条件叠加实验**（Phase 2）。每组实验均包含多次重复试验以量化一致性。

## 二、实验对象

### 2.1 被测模型

建议覆盖以下类别，具体模型根据实际可用性调整：

请求格式是：{"model":"glm-5.1","messages":[{"role":"user","content":"你好"}]}

### 2.2 采样参数

每组实验对每个模型设置两种采样条件：

- **贪心解码**（temperature=0）：测试模型的"确定性偏好"
- **随机采样**（temperature=0.7）：测试模型的"偏好分布"

## 三、Phase 1：理想环境实验

### 3.1 设计哲学

理想环境下，所有坑位仅在位置上有差异，没有任何额外信息（无脏乱、无气味、无人）。这是最纯粹的决策场景——模型的选择完全来自它对"位置"本身的偏好，或者来自它训练数据中关于此类场景的统计规律。

### 3.2 实验矩阵

#### 实验1.1：基准选择（全空坑位）

**场景设定**：一排 N 个坑位，全部空闲，无任何附加信息。

| 变量 | 取值 |
|------|------|
| 坑位数量 N | 3, 5, 7, 10 |
| 重复次数 | 每个 N × 每个模型 × 每种采样 = 30次 |
| Prompt模板 | 3种不同表述（见3.3节） |

**核心观测**：

- 模型是否倾向于选择端点位置（位置1或位置N）？
- 模型是否倾向于选择中间位置？
- 选择分布是均匀的还是有明显偏好？
- 30次重复中，选择的集中度如何？

#### 实验1.2：对称性检验

**场景设定**：场景完全对称，检验模型是否能识别对称性。

| 测试项 | 具体内容 |
|--------|---------|
| 左右对称 | 5个坑位全空，问模型选哪个。若模型理解对称性，对坑位1和坑位5的偏好应无显著差异 |
| 编号反转 | 同一排坑位，一次从左到右编号（1-5），一次从右到左编号（5-1），选择是否翻转 |
| 方向描述 | "从入口数第1个" vs "最里面的那个"，指向同一坑位时选择是否一致 |

**核心观测**：

- 模型是否有"编号偏好"（比如偏好数字小的选项）而非真正的位置偏好？
- 反转编号后选择是否跟着翻转——如果翻转，说明模型关注编号而非位置语义

#### 实验1.3：选项数量效应

**场景设定**：逐步增加坑位数量（3→5→7→10→15→20），保持全空，观察模型的选择策略是否有结构性变化。

**核心观测**：

- 是否存在一个阈值，超过该阈值后模型的选择策略发生质变？
- 模型选择的相对位置（选第几个/总数）是否在不同N下保持稳定？
- 选项数量增加是否导致一致性下降？

### 3.3 Prompt模板设计

为检验表述敏感性，每个场景使用至少3种语义等价但措辞不同的prompt：

**模板A（简洁直述型）**：
"一排N个坑位，全部空闲。你选第几个？只回答数字。"

**模板B（场景描述型）**：
"你走进一间公共厕所，面前有一排N个独立的坑位，从左到右依次编号1到N。现在所有坑位都没有人使用。你会选择哪一个坑位？请直接回答你的选择。"

**模板C（第三人称型）**：
"一个人走进一间公共厕所，看到一排N个空坑位。从理性决策的角度，这个人应该选择哪一个坑位？请给出你的建议。"

**模板D（无编号型）**：
"面前有一排N个坑位，全部空闲。你可以选最左边的、最右边的、中间的、或者靠左/靠右的某一个。你怎么选？"

### 3.4 Phase 1 预期产出

- 每个模型在理想环境下的**选择分布热力图**（横轴：坑位位置，纵轴：模型）
- **一致性得分**：30次重复中最高频选择的占比（1.0=完全一致，1/N=完全随机）
- **对称性违反率**：在对称场景中，左右对称位置的选择频率差异
- **表述敏感性指标**：不同prompt模板下选择分布的KL散度

## 四、Phase 2：条件叠加实验

### 4.1 设计哲学

在理想环境的基础上，逐步叠加单一条件变量，观察模型的决策如何响应。每次只改变一个条件，保持其他条件与Phase 1一致。这样可以隔离每个条件对决策的影响。

### 4.2 条件变量设计

#### 条件A：占用状态（Occupancy）

基础设定：5个坑位。

| 编号 | 占用情况 | 检验目的 |
|------|---------|---------|
| A1 | 坑位3被占用 | 对称占用下的偏好 |
| A2 | 坑位1被占用 | 端点被占后是否转向另一端点 |
| A3 | 坑位2和坑位4被占用 | 多人占用下的社交距离计算 |
| A4 | 坑位1和坑位5被占用 | 两端被占，是否选中间 |
| A5 | 坑位1、2、4被占用 | 高拥挤下的选择（仅剩3和5） |
| A6 | 仅坑位3空闲 | 无选择余地，模型是否直接选3 |

每组重复30次 × 3种prompt模板。

**核心观测**：

- 模型是否表现出明确的"社交距离最大化"策略？
- 对称占用场景（如A1）下，模型的选择是否对称？
- 占用人数增加时，一致性是否下降？

#### 条件B：清洁度信息（Cleanliness）

基础设定：5个坑位全空，但部分坑位有清洁度描述。

| 编号 | 清洁度信息 | 检验目的 |
|------|-----------|---------|
| B1 | 坑位3地面有水渍 | 单一负面信息的影响 |
| B2 | 坑位1特别干净 | 单一正面信息的影响 |
| B3 | 坑位1有异味，坑位5地面潮湿 | 两端均有负面信息 |
| B4 | 所有坑位都有轻微使用痕迹，程度相同 | 无差异负面信息是否影响选择策略 |
| B5 | 坑位3刚打扫过 | 正面信息是否能克服中间位置的隐私劣势 |

**核心观测**：

- 清洁度信息是否会压过位置偏好？
- 当清洁度和位置偏好冲突时，模型如何权衡？
- 权衡方式在多次试验中是否一致？

#### 条件C：时间压力（Time Pressure）

基础设定：5个坑位全空，但增加时间紧迫性描述。

| 编号 | 时间描述 | 检验目的 |
|------|---------|---------|
| C1 | "你非常着急" | 是否转向选择最近的（位置1） |
| C2 | "你不赶时间，可以慢慢选" | 是否与无时间描述的基准一致 |
| C3 | "你有点急但还能忍一下" | 中间程度的时间压力 |

**核心观测**：

- 时间压力是否改变了位置偏好？
- 如果改变，模型是否表现出合理的权衡（急→选近，不急→选好）？
- C2与Phase 1基准的选择是否一致（应该一致）？

#### 条件D：社交情境（Social Context）

基础设定：5个坑位，叠加社交信息。

| 编号 | 社交情境 | 检验目的 |
|------|---------|---------|
| D1 | "厕所外面还有人排队" | 外部社会压力的影响 |
| D2 | "坑位2有人，而且在打电话" | 噪音因素的引入 |
| D3 | "坑位4有人，门没关好" | 视觉隐私因素的引入 |
| D4 | "你和朋友一起来的，他选了坑位1" | 熟人关系的影响 |

**核心观测**：

- 模型是否理解不同社交因素的含义并做出合理响应？
- 社交情境的引入是否显著降低选择一致性？

#### 条件E：物理环境差异（Physical Differences）

基础设定：5个坑位，有物理差异。

| 编号 | 物理差异 | 检验目的 |
|------|---------|---------|
| E1 | "坑位5旁边有窗户" | 通风/光线因素 |
| E2 | "坑位1门锁有点松" | 功能缺陷因素 |
| E3 | "坑位3空间比其他的大一些" | 舒适性因素 |
| E4 | "坑位1是蹲坑，其余是马桶" | 设施类型偏好 |

**核心观测**：

- 模型是否将物理差异纳入决策？
- 纳入的方式是否合理？
- 不同物理差异的优先级排序在多次试验中是否稳定？

### 4.3 条件组合实验

在单一条件实验完成后，选择关键条件进行两两组合，检验模型在多维度决策下的表现。

| 编号 | 组合条件 | 检验目的 |
|------|---------|---------|
| X1 | A3（坑位2、4有人）+ B1（坑位3有水渍）| 社交距离 vs 清洁度的权衡 |
| X2 | A2（坑位1有人）+ C1（非常着急）| 社交距离 vs 时间压力的权衡 |
| X3 | B5（坑位3刚打扫过）+ E1（坑位5有窗户）| 清洁度 vs 通风的权衡 |
| X4 | A3 + C1 + B1 | 三条件叠加——高复杂度决策 |

**核心观测**：

- 当维度冲突时，模型是否有稳定的优先级排序？
- 多次试验中，权衡方式是否一致？
- 条件组合后一致性是否进一步下降？

### 4.4 公理专项测试

#### 传递性测试

利用Phase 2的结果构造传递性检验：

**设计方式**：

- 从条件实验中提取三组两两比较场景
- 例如：场景α下模型偏好坑位5 > 坑位1；场景β下偏好坑位1 > 坑位3
- 构造场景γ（同等条件下坑位5 vs 坑位3的选择），检验是否坑位5 > 坑位3
- 构造至少20组这样的传递性三元组

**报告指标**：传递性违反率 $V_T = \frac{\text{违反三元组数}}{总三元组数}$

#### IIA测试

**设计方式**：

- 场景1：3个坑位（1, 3, 5）全空，模型选了坑位X
- 场景2：增加坑位2和坑位4（全空），变成5个坑位全空
- 检验：坑位X在场景2中是否仍然被选择（或至少仍是最优选之一）

**报告指标**：IIA违反率 $V_{IIA}$

#### 框架效应测试

**设计方式**：

- 正面框架："坑位1特别干净"
- 负面框架："坑位2、3、4、5不太干净"（语义等价——只有1干净）
- 检验两种框架下的选择是否一致

**报告指标**：框架效应强度 $F = |P_{\text{positive}} - P_{\text{negative}}|$

## 五、数据收集规范

### 5.1 数据采集流程

```
准备阶段：
  1. 确定被测模型清单及API接入方式
  2. 生成全部prompt（场景 × 模板 × 重复次数）
  3. 为每条prompt分配唯一ID
  4. 对prompt顺序进行随机化（避免顺序效应）

执行阶段：
  5. 每条prompt作为独立的新对话发送（不携带上文）
  6. 记录模型的完整回复原文
  7. 从回复中提取选择结果（坑位编号）
  8. 若模型拒绝选择或给出无法解析的回答，标记为 "REFUSED" 或 "AMBIGUOUS"
  9. 记录API返回的元信息（token数、延迟等）

质控阶段：
  10. 对全部 "AMBIGUOUS" 回复进行人工判读
  11. 抽检5%的正常回复，验证自动提取的准确性
  12. 检查是否有API错误或超时导致的缺失数据
```

## 六、数据处理与分析

### 6.1 数据清洗

**步骤1：选择提取验证**

- 自动提取与人工抽检的一致率应 ≥ 98%
- 对 AMBIGUOUS 和 REFUSED 回复单独统计，分析是否与特定场景/模型/模板相关
- 若某模型的 REFUSED 率 > 20%，该模型的该组数据可能不适合纳入主分析，但 REFUSED 率本身是一个有价值的发现

**步骤2：异常值处理**

- 检查是否有模型返回不在有效选项中的坑位编号（如5个坑位选了6）
- 这类回复标记为 INVALID，统计其比例

### 6.2 核心分析指标

#### 指标1：选择一致性（Choice Consistency）

对每个实验组内同一条件下的30次重复，计算：

**众数一致率（Mode Consistency Rate, MCR）**：
$$MCR = \frac{\text{最高频选择的出现次数}}{总重复次数}$$

- MCR = 1.0：完美一致（30次都选同一个）
- MCR = 1/N：等同于随机选择
- 中间值：有偏好但不完全稳定

**选择熵（Choice Entropy）**：
$$H = -\sum_{i=1}^{N} p_i \log_2 p_i$$

其中 $p_i$ 是坑位 $i$ 被选中的频率。H越低，选择越集中。

#### 指标2：跨模板一致性（Cross-Template Consistency）

对同一场景的不同prompt模板，计算选择分布之间的Jensen-Shannon散度：

$$JSD(P_A \| P_B) = \frac{1}{2} KL(P_A \| M) + \frac{1}{2} KL(P_B \| M), \quad M = \frac{P_A + P_B}{2}$$

JSD = 0 表示两个模板下选择分布完全相同，JSD越大表示表述敏感性越强。

#### 指标3：条件响应合理性（Condition Response Rationality）

当条件变化时，模型的选择变化是否"合理"。需要预先定义"合理响应"的标准：

| 条件变化 | 合理响应 | 不合理响应 |
|---------|---------|-----------|
| 增加坑位占用 → | 远离被占坑位 | 靠近被占坑位或无变化 |
| 增加时间压力 → | 选择更靠近入口的 | 选择更远的 |
| 某坑位标记为脏 → | 避开该坑位 | 反而选择该坑位 |
| 某坑位标记为干净 → | 倾向于选该坑位 | 反而避开该坑位 |

对每个条件变化，计算**合理响应率**：在所有该条件相关的试验中，模型做出合理响应的比例。

#### 指标4：公理违反率

- $V_T$：传递性违反率
- $V_{IIA}$：IIA违反率
- $F$：框架效应强度
- $DCI$：综合决策一致性指标（见理论框架文档中的定义）

#### 指标5：隐含效用函数可重构性

**方法**：对每个模型的全部Phase 2数据，尝试用最大似然估计拟合一个多属性效用函数：

$$U(a_i) = \hat{w}_1 \cdot f_{\text{privacy}}(a_i) + \hat{w}_2 \cdot f_{\text{distance}}(a_i) + \hat{w}_3 \cdot f_{\text{clean}}(a_i) + \hat{w}_4 \cdot f_{\text{convenience}}(a_i)$$

如果拟合优度良好（模型的实际选择能被该效用函数高比例地预测），说明模型"好像"有一个内部效用函数。如果拟合优度差，说明模型的决策无法被一个稳定的效用函数解释。

### 6.3 统计检验

所有核心结论需配合统计检验：

| 检验目的 | 推荐方法 |
|---------|---------|
| 选择分布是否均匀（有无偏好）| 卡方拟合优度检验 |
| 两种prompt下选择分布是否一致 | 卡方独立性检验 / Fisher精确检验 |
| 编号反转后选择是否翻转 | 配对符号检验（McNemar检验）|
| 条件变化前后选择是否显著变化 | McNemar检验 / 配对卡方检验 |
| 不同模型间一致性得分的差异 | Kruskal-Wallis检验 + Dunn事后检验 |
| 效用函数拟合优度 | 对数似然比检验 / AIC/BIC比较 |

显著性水平设为 α = 0.05，对多重比较进行 Bonferroni 校正。

## 七、结果撰写指南

### 7.1 结果部分的结构

论文的Results部分建议按以下顺序组织，每个小节对应一个核心发现：

#### 7.1.1 发现一：大模型存在位置偏好，但偏好因模型而异

报告Phase 1的选择分布。预期发现：大部分模型在全空场景下有非均匀的选择分布（比如偏好端点或偏好靠前位置），但不同模型的偏好位置不同。这说明模型的选择不是随机的，但也不是来自统一的决策逻辑。

**关键数据呈现**：

- Phase 1全空场景的选择热力图（核心图表）
- 各模型MCR得分表
- 卡方检验结果：选择分布是否显著偏离均匀分布

#### 7.1.2 发现二：选择一致性在贪心解码下较高但仍不完美

报告MCR和选择熵在不同采样条件下的差异。预期发现：temperature=0时MCR较高（但可能仍<1.0，因为部分模型即使在贪心解码下也可能因输入的微小差异产生不同输出），temperature=0.7时MCR显著下降。

**关键数据呈现**：

- 贪心 vs 随机采样的MCR对比表
- 选择熵的分布直方图

#### 7.1.3 发现三：选择对表述方式高度敏感

报告跨模板一致性（JSD）。预期发现：同一场景在不同prompt模板下的选择分布存在显著差异，JSD > 0。这直接说明模型的决策与表述耦合。

**关键数据呈现**：

- 各模板对之间的JSD矩阵
- 典型案例分析：展示一两个模型在不同模板下选择完全翻转的具体例子

#### 7.1.4 发现四：模型对编号而非语义敏感

报告Phase 1对称性检验结果。预期发现：编号反转后，部分模型的选择跟着编号走而非跟着位置走，说明模型在做的不是"空间推理"而是"编号匹配"。

**关键数据呈现**：

- 编号正向 vs 反向时各位置的选择频率对比
- McNemar检验结果

#### 7.1.5 发现五：条件变化能引发选择变化，但权衡方式不一致

报告Phase 2的条件响应合理性和一致性。预期发现：模型在多数情况下能对条件变化做出"方向正确"的响应（如避开脏的坑位），但在多次试验中权衡的具体结果不一致。

**关键数据呈现**：

- 合理响应率汇总表（条件类型 × 模型）
- 条件叠加后MCR的变化趋势图
- 典型权衡冲突案例分析

#### 7.1.6 发现六：大模型系统性违反决策理性公理

报告公理专项测试结果。预期发现：传递性违反率和IIA违反率均显著大于零，框架效应普遍存在。

**关键数据呈现**：

- 各模型的 $V_T$, $V_{IIA}$, $F$, $DCI$ 汇总表（核心表格）
- 与人类决策数据的对比（如可获得）

#### 7.1.7 发现七：隐含效用函数无法稳定重构

报告效用函数拟合结果。预期发现：模型的选择数据无法被一个稳定的线性效用函数很好地解释，拟合优度较低或在不同子集上拟合出的权重差异巨大。

**关键数据呈现**：

- 各模型的效用函数拟合优度（预测准确率）
- 拟合出的隐含权重及其置信区间
- 子集间权重稳定性分析

### 7.2 写作注意事项

**客观性**：报告发现时保持中性，避免过度渲染"LLM决策很差"。客观呈现数据，让数据说话。

**完整性**：如果某些结果与预期不符（比如某个模型的一致性出乎意料地高），如实报告并讨论可能原因，不要回避。

**可复现性**：在附录中提供完整的prompt列表、实验参数、随机种子（如适用）、数据处理脚本的说明。

**讨论的衔接**：Results部分末尾应自然过渡到Discussion，引出"这些缺陷是结构性的→需要新的决策范式→引出CSHDA"的叙事线。

### 8.2 风险与预案

| 风险 | 影响 | 预案 |
|------|------|------|
| 某模型API不稳定或被限速 | 数据缺失 | 设置重试机制；准备替代模型 |
| 模型拒绝回答（"我不会上厕所"） | REFUSED比例过高 | 调整prompt（如改为第三人称/建议视角） |
| 预算超支 | 无法测试所有模型 | 先跑核心模型子集，确认发现后再扩展 |
| Phase 1结果与预期完全不符 | 论文叙事需调整 | 如实报告，作为有价值的"负面结果"也可发表 |
| 选择提取困难（模型回复冗长或格式不一） | 数据处理耗时 | 在prompt中强制要求格式化输出（如"只回答数字"） |
